# Telegram Support Bot

> Этот проект представляет собой Telegram-бота для организации простейшей системы поддержки.
> Бот принимает сообщения пользователей (текст, фото, видео, аудио, документы, голосовые) и пересылает их в чат поддержки.
> Операторы поддержки отвечают на сообщения прямо в топике пользователя, а бот автоматически перенаправляет ответ тому пользователю, от которого пришёл запрос.

*Это мой первый публичный репозиторий, поэтому не ругайте строго ✨*

## 🚀 Возможности

* **🔄 Двусторонняя связь:** Пересылка сообщений от пользователей в чат поддержки и обратно.
* **📁 Персональные топики:** Автоматическое создание отдельного топика для каждого пользователя с отображением статуса (🟢 открыт / 🔴 закрыт). При повторном обращении топик переиспользуется.
* **⛔️ Блокировка пользователей**: Инлайн-кнопки для бана/разбана пользователей прямо в интерфейсе чата.
* **🎫 Система тикетов**: Автоматическое создание уникального ID тикета для каждого пользователя.
* **📊 Управление статусами**: open/closed, команды `/close`, `/reopen`.
* **📈 Список тикетов**: `/open_tickets` — все открытые тикеты с деталями.
* **📤 Поддержка всех медиа**: Текст, фото, видео, GIF, документы, голосовые, аудио, видеосообщения, стикеры, контакты, геолокация. Форматирование и premium emoji сохраняются.
* **💾 База данных:** SQLite для хранения связей сообщений, тикетов и чёрного списка.

## 📋 Требования

* Python 3.9+
* Токен Telegram бота
* Супергруппа с включенными топиками (темы)
* Бот должен быть администратором группы с правами `Управление темами`

## 🛠 Установка

### 1. Подготовка Telegram

1. Напишите [@BotFather](https://t.me/BotFather), создайте бота и скопируйте токен.
2. Создайте группу (или используйте существующую), включите топики: **Настройки группы → Темы**.
3. Добавьте бота в группу как администратора с правом **«Управление темами»**.
4. Узнайте ID группы — перешлите любое сообщение из неё боту [@userinfobot](https://t.me/userinfobot). ID начинается с `-100`.

### 2. Настройка конфигурации

```bash
git clone https://github.com/v1rtezz/support-bot.git
cd support-bot
cp .env.example .env
nano .env
```

Заполните `.env`:

```ini
TELEGRAM_TOKEN=123456789:AAH...        # Токен от BotFather
SUPPORT_CHAT_ID=-1001234567890         # ID группы поддержки 
```

### 3. Деплой на VPS (Ubuntu/Debian)

```bash
apt install -y python3.12-venv    # если ещё не установлен
./deploy.sh
```

Скрипт автоматически:
- Создаст виртуальное окружение и установит зависимости
- Настроит systemd-сервис `support-bot`
- Запустит бота в фоне с автозапуском после перезагрузки

### 4. Управление ботом

| Команда | Описание |
|---------|----------|
| `sudo systemctl status support-bot` | Статус бота |
| `sudo journalctl -u support-bot -f` | Логи в реальном времени |
| `sudo systemctl restart support-bot` | Перезапуск (после изменения `.env` или кода) |
| `sudo systemctl stop support-bot` | Остановка |

Для обновления кода:

```bash
git pull && ./deploy.sh
```

### Локальный запуск (для разработки)

```bash
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python3 bot.py
```

## 📖 Как использовать

### Для пользователей

1. Пользователь запускает бота командой `/start`.
2. Пишет сообщение, отправляет фото или видео.
3. Бот создаёт персональный топик в группе поддержки и пересылает туда сообщение.

> Если пользователь заблокирован, бот будет игнорировать его сообщения.

### Для операторов поддержки

1. Для каждого нового пользователя создаётся отдельный топик с названием «🟢 Username» или «🟢 Имя».
2. В первом сообщении топика — информация о пользователе (ID, username, имя).
3. Все последующие сообщения от этого пользователя приходят в этот же топик.
4. Оператор может ответить через Reply или просто написать в топик — бот перешлёт ответ пользователю.
5. При закрытии тикета название топика меняется на «🔴 Username».
6. Под каждым сообщением есть кнопка **❌ Заблокировать/Разблокировать**.

## 🤖 Команды бота

### Для пользователя (в ЛС бота)

* `/start` — приветственное сообщение и инструкция.
* `/help` — подсказка по работе поддержки.

### Для операторов (в чате поддержки)

* `/open_tickets` — список открытых тикетов с деталями.
* `/close` (по реплаю) — закрывает тикет и уведомляет пользователя.
* `/reopen` (по реплаю) — открывает тикет заново.
* `/ticket` (по реплаю) — информация о тикете (ID, статус, блокировка, даты).
